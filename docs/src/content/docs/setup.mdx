Wyck is a custom biome API for [Paper](https://papermc.io/software/paper) servers without the use of datapacks.
Our wrappers allow you to easily add new biomes to the game and send them to clients without touching any internals.

## Compatibility

Wyck is compatible with **Minecraft 1.21.11, 26.1.2, 26.2** and requires a [Paper](https://papermc.io/software/paper) server running on **Java 25** or later.
Wyck is also compatible with [Folia](https://papermc.io/software/folia), but it is up to you to ensure that the rest of your plugin is compatible with Folia if you choose to support it.


## Installation

Installation depends on your build system and whether you choose to use Wyck as a shaded dependency or external plugin dependency.
This guide will assume you are using **Gradle (Kotlin DSL)** and will cover both shaded and external dependency methods.

The latest stable release is [<Versions />](https://github.com/LumaLibre/Wyck/releases) and the latest snapshot is [<Versions channel="snapshot" />](https://repo.wyck.dev/#/releases/dev/wyck/Wyck).

<CodeTabs
    examples={[
        {
            label: "Shaded",
            filename: "build.gradle.kts",
            lang: "kotlin",
            code:
`plugins {
    id("com.gradleup.shadow") version "9.4.2"
}

repositories {
    maven("https://repo.wyck.dev/public/")
}

dependencies {
    implementation("dev.wyck:Wyck:${latestRelease}")
}

tasks {
    shadowJar {
        val pack = "com.example.testplugin"
        relocate("dev.wyck", "\$pack.wyck")
        archiveClassifier.set("") // Remove the -all suffix
    }

    jar {
        enabled = false // Disable the default jar task
    }

    build {
        dependsOn(shadowJar) // Ensure the shadowJar task is run before the build
    }
}`,
        },
        {
            label: "External",
            filename: "build.gradle.kts",
            lang: "kotlin",
            code:
`repositories {
    maven("https://repo.wyck.dev/public/")
}

dependencies {
    compileOnly("dev.wyck:Wyck:${latestRelease}")
}`,
        },
    ]}
/>

**If you are using Wyck as external dependency** you will need to add Wyck as a plugin dependency in your `plugin.yml`, or `paper-plugin.yml`.

<CodeTabs
    examples={[
        {
            label: "Bukkit Plugin",
            filename: "plugin.yml",
            lang: "yaml",
            code:
`name: ExamplePlugin
main: com.example.ExamplePlugin
version: 1.0-SNAPSHOT
api-version: 1.21
depend: [Wyck] # Add this line!`,
        },
        {
            label: "Paper Plugin",
            filename: "paper-plugin.yml",
            lang: "yaml",
            code:
`name: ExamplePlugin
main: com.example.ExamplePlugin
bootstrapper: com.example.ExampleBootstrapper
version: 1.0-SNAPSHOT
api-version: 1.21
# Add these lines:
dependencies:
    bootstrap:
      Wyck:
        load: BEFORE
        required: true
        join-classpath: true
    server:
      Wyck:
        load: BEFORE
        required: true
        join-classpath: true`,
        },
    ]}
/>

Wyck external dependency plugin can be found on our GitHub releases or [Modrinth](https://modrinth.com/plugin/biomesapi).
